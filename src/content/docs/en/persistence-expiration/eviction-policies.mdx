## Setting a memory limit

`maxmemory` caps Redis memory usage. When the limit is reached, Redis applies the eviction policy to decide which keys to remove. You can set it in `redis.conf` or at runtime with `CONFIG SET`.

```ini
maxmemory 512mb
maxmemory-policy allkeys-lru
```

## Eviction policies

| Policy | Behavior |
|--------|----------|
| `noeviction` | Return error on writes when full (default; safe for databases) |
| `allkeys-lru` | Evict least recently used key from all keys |
| `volatile-lru` | Evict LRU from keys with a TTL set |
| `allkeys-lfu` | Evict least frequently used key from all keys |
| `volatile-lfu` | Evict LFU from keys with a TTL |
| `volatile-ttl` | Evict the key with the shortest remaining TTL |
| `allkeys-random` | Evict a random key from all keys |
| `volatile-random` | Evict a random key from keys with a TTL |

## Checking and changing policy at runtime

```text
127.0.0.1:6379> CONFIG GET maxmemory-policy
1) "maxmemory-policy"
2) "noeviction"
127.0.0.1:6379> CONFIG SET maxmemory-policy allkeys-lru
OK
127.0.0.1:6379> CONFIG GET maxmemory
1) "maxmemory"
2) "0"
127.0.0.1:6379> CONFIG SET maxmemory 256mb
OK
```

<Callout title="Cache vs database — choose the right policy">

As a cache, set `maxmemory` plus an `allkeys-lru` or `allkeys-lfu` policy so Redis automatically removes the least useful keys. As a primary database, use `noeviction` so writes fail loudly rather than silently dropping data.

</Callout>
