## การกำหนดขีดจำกัดหน่วยความจำ

`maxmemory` จำกัดการใช้หน่วยความจำของ Redis เมื่อถึงขีดจำกัด Redis จะใช้นโยบาย eviction เพื่อตัดสินใจว่าจะลบ key ใด คุณสามารถตั้งค่าได้ใน `redis.conf` หรือที่ runtime ด้วย `CONFIG SET`

```ini
maxmemory 512mb
maxmemory-policy allkeys-lru
```

## นโยบาย Eviction

| นโยบาย | พฤติกรรม |
|--------|----------|
| `noeviction` | คืนค่า error เมื่อเขียนและหน่วยความจำเต็ม (ค่าเริ่มต้น; ปลอดภัยสำหรับฐานข้อมูล) |
| `allkeys-lru` | ลบ key ที่ใช้งานน้อยที่สุดล่าสุดจากทุก key |
| `volatile-lru` | ลบ LRU จาก key ที่มี TTL กำหนด |
| `allkeys-lfu` | ลบ key ที่ใช้งานน้อยที่สุดจากทุก key |
| `volatile-lfu` | ลบ LFU จาก key ที่มี TTL |
| `volatile-ttl` | ลบ key ที่มี TTL เหลือน้อยที่สุด |
| `allkeys-random` | ลบ key แบบสุ่มจากทุก key |
| `volatile-random` | ลบ key แบบสุ่มจาก key ที่มี TTL |

## การตรวจสอบและเปลี่ยนนโยบายที่ runtime

```text
127.0.0.1:6379> CONFIG GET maxmemory-policy
1) "maxmemory-policy"
2) "noeviction"
127.0.0.1:6379> CONFIG SET maxmemory-policy allkeys-lru
OK
127.0.0.1:6379> CONFIG GET maxmemory
1) "maxmemory"
2) "0"
127.0.0.1:6379> CONFIG SET maxmemory 256mb
OK
```
