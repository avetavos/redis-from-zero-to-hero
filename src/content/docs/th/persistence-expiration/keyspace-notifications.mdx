## Keyspace Notifications คืออะไร?

Redis สามารถ publish ข้อความ Pub/Sub ได้ทุกครั้งที่เกิด event บางอย่าง — เช่น key หมดอายุ, มีการรัน `SET`, มีการรัน `DEL` และอื่น ๆ แอปพลิเคชันของคุณ subscribe ช่องทางเหล่านี้เพื่อตอบสนองโดยไม่ต้อง polling: ตัวอย่างเช่น ล้าง local cache เมื่อ key หมดอายุ หรือเรียก background job เมื่อมีการเขียน key เฉพาะ

Redis เปิดเผยช่องทาง 2 ประเภท:

- **Keyevent channels** — หนึ่งช่องทางต่อประเภท event; payload ของข้อความคือชื่อ key รูปแบบ: `__keyevent@<db>__:<event>` (ตัวอย่าง: `__keyevent@0__:expired`)
- **Keyspace channels** — หนึ่งช่องทางต่อ key; payload ของข้อความคือประเภท event รูปแบบ: `__keyspace@<db>__:<keyname>`

## การเปิดใช้งาน Keyspace Notifications

Keyspace notifications **ถูกปิดใช้งานโดยค่าเริ่มต้น** เนื่องจากการ publish ทุก event จะเพิ่มภาระ CPU เปิดใช้งานด้วย `CONFIG SET` ขณะรันไทม์ หรือผ่าน `redis.conf` เมื่อเริ่มต้น

ค่าที่กำหนดคือสตริงของอักขระ flag ที่เลือกว่าจะ publish event class ใดบ้าง:

| Flag | Events ที่เปิดใช้งาน |
|------|---------------------|
| `K` | Keyspace events (ช่องทาง `__keyspace@...`) |
| `E` | Keyevent events (ช่องทาง `__keyevent@...`) |
| `x` | Expired events (TTL ของ key ถึงศูนย์) |
| `g` | คำสั่งทั่วไป: `DEL`, `EXPIRE`, `RENAME`, ... |
| `s` | คำสั่ง Set (`SADD`, `SREM`, ...) |
| `l` | คำสั่ง List (`LPUSH`, `RPOP`, ...) |

สำหรับ expiry notifications ใน production การตั้งค่าที่นิยมใช้คือ `Ex` — keyevent channel (`E`) สำหรับ expired events เท่านั้น (`x`):

```ini
notify-keyspace-events "Ex"
```

หรือขณะรันไทม์:

```text
127.0.0.1:6379> CONFIG SET notify-keyspace-events Ex
OK
```

## การ Subscribe รับ expiry events

Keyspace notifications ใช้การทำงานแบบสอง terminal เปิด **Terminal 1** และ subscribe ก่อนที่ key จะหมดอายุ:

```text
127.0.0.1:6379> PSUBSCRIBE __keyevent@0__:expired
Reading messages... (press Ctrl-C to quit)
```

ใน **Terminal 2** กำหนด key ที่มี TTL สั้น ๆ เพื่อ trigger event:

```text
127.0.0.1:6379> SET demo:token "abc" EX 3
OK
```

หลังจาก 3 วินาที Terminal 1 จะได้รับการแจ้งเตือน:

```text
1) "pmessage"
2) "__keyevent@0__:expired"
3) "__keyevent@0__:expired"
4) "demo:token"
```

บรรทัดที่สี่คือชื่อ key — ตอนนี้ subscriber ของคุณรู้แน่ชัดว่า key ไหนหมดอายุ

## คำสั่งที่รันได้

คำสั่งด้านล่างเปิดใช้งาน notifications และกำหนด key ที่มี TTL เพื่อให้คุณสังเกตการกำหนดค่าในการทำงาน หากต้องการดู expired event ให้เปิด `redis-cli` ตัวที่สอง รัน `PSUBSCRIBE __keyevent@0__:expired` จากนั้นใน terminal แรกรัน `SET demo:token "hello" EX 5` แล้วรอ 5 วินาที
