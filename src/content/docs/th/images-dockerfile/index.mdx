## Docker image คืออะไร?

Docker **image** คือแพ็กเกจแบบ read-only ที่บรรจุทุกอย่างที่จำเป็นต่อการรันโปรแกรม ไม่ว่าจะเป็น filesystem ของ OS, runtime, ไลบรารี, โค้ดแอปพลิเคชัน และค่า config เริ่มต้น ลองนึกถึงมันเหมือนสูตรอาหาร ตัวสูตรไม่เคยเปลี่ยน แต่คุณสามารถอบเค้ก (container) จากสูตรนี้ได้มากเท่าที่ต้องการ

image ถูกสร้างขึ้นจาก **layer** แบบ read-only ที่ซ้อนกันเป็นชั้น ๆ แต่ละ layer แทนการเปลี่ยนแปลงหนึ่งอย่าง เช่น เพิ่มไฟล์, ติดตั้งแพ็กเกจ, หรือเขียนค่า config เมื่อนำ layer มาซ้อนกันก็จะได้ filesystem ที่สมบูรณ์ แต่ Docker จะเก็บ (และส่งต่อ) เฉพาะ layer ที่มีการเปลี่ยนแปลงเท่านั้น จึงทำให้ image มีขนาดเล็กกระชับ

```text
Image (read-only)
├── Layer 3: COPY app/ /app/          ← your code
├── Layer 2: RUN npm install          ← dependencies
├── Layer 1: FROM node:22-alpine      ← base OS + runtime
```

**container** ที่กำลังรันอยู่ก็คือ image ที่มี writable layer เพิ่มขึ้นมาอีกหนึ่งชั้นด้านบน ตัว image ไม่ถูกแตะต้อง container จะเขียนข้อมูลลงใน scratch layer ของตัวเอง เมื่อหยุด container layer นั้นก็จะหายไป (เว้นแต่คุณจะบันทึกเอาไว้)

```text
Container (running)
├── Writable layer  ← container-specific changes (deleted on stop)
├── Layer 3 (read-only)
├── Layer 2 (read-only)
└── Layer 1 (read-only)
```

## Dockerfile คืออะไร?

**Dockerfile** คือไฟล์ข้อความธรรมดาที่บรรจุชุดคำสั่ง (instruction) เพื่อบอก Docker ว่าจะ build image ขึ้นมาอย่างไร แต่ละ instruction จะกลายเป็น layer หนึ่งชั้น คุณรัน `docker build` เพื่อเปลี่ยน Dockerfile ให้เป็น image และรัน `docker run` เพื่อสตาร์ต container จาก image นั้น

นี่คือ Dockerfile ที่ง่ายที่สุดเท่าที่จะเป็นไปได้:

```dockerfile
# syntax=docker/dockerfile:1
FROM alpine:3.20
CMD ["echo", "Hello, Docker!"]
```

build และรัน:

```bash
docker build -t hello .
docker run --rm hello
```

ผลลัพธ์ที่คาดหวัง:

```text
Hello, Docker!
```

## โมดูลนี้ครอบคลุมเรื่องอะไรบ้าง

| บทเรียน | สิ่งที่คุณจะได้เรียนรู้ |
|---|---|
| หน้านี้ | image กับ container, layer, การ build ครั้งแรก |
| Dockerfile basics | instruction หลัก: FROM, WORKDIR, COPY, RUN, ENV, EXPOSE |
| Build & tag | build context, `-t name:tag`, `docker images`, `.dockerignore` |
| Layers & cache | build cache ทำงานอย่างไร, การจัดลำดับ instruction ให้ build เร็ว |
| CMD vs ENTRYPOINT | exec form, shell form, default arguments, การ override |
