## Container คืออะไร?

Image คือพิมพ์เขียว ส่วน **container** คือ instance ที่กำลังทำงานอยู่จริงของพิมพ์เขียวนั้น — เป็นกระบวนการที่ถูกแยกออกมา มีระบบไฟล์, เครือข่าย, และ process space เป็นของตัวเอง โดยทั้งหมดใช้ host kernel ร่วมกัน

คุณสามารถรัน container หลายตัวจาก image เดียวกัน, หยุดและรีสตาร์ทได้, ดู log, และลบทิ้งเมื่อใช้งานเสร็จ การเข้าใจวงจรชีวิตนี้เป็นพื้นฐานสำคัญของการทำงานกับ Docker ในชีวิตประจำวัน

## วงจรชีวิตของ container

Container ผ่านสถานะทั้งสี่นี้:

| สถานะ | ความหมาย |
|-------|----------|
| **Created** | `docker create` จัดสรร container ไว้แล้ว แต่ยังไม่เริ่มทำงาน |
| **Running** | กระบวนการหลักของ container กำลังทำงานอยู่ |
| **Stopped** | กระบวนการหลักสิ้นสุดแล้ว (หรือคุณเรียก `docker stop`) — container ยังอยู่บนดิสก์ |
| **Removed** | `docker rm` ลบ container และคืนพื้นที่ writable layer |

```text
docker run   →  Created + Running
docker stop  →  Stopped
docker start →  Running again
docker rm    →  Removed (gone)
```

## สิ่งที่จะเรียนในโมดูลนี้

| บทเรียน | หัวข้อ |
|---------|--------|
| **Lifecycle** | `docker run`, `docker ps`, `docker stop/start/restart`, `docker rm`, `--rm` |
| **Exec & Logs** | `docker exec -it`, `docker logs`, `docker top`, การ attach |
| **Ports & Env** | เผยแพร่พอร์ตด้วย `-p`, ตัวแปร environment ด้วย `-e`, การตั้งชื่อด้วย `--name` |
| **Restart & Inspect** | นโยบาย restart, `docker inspect`, `docker stats`, การจำกัด resource |

## ตัวอย่างด่วน — รัน, แสดงรายการ, และหยุด

สามคำสั่งที่คุณจะใช้บ่อยที่สุดในโมดูลนี้:

```bash
# Pull และรัน nginx ในเบื้องหลัง
docker run -d --name demo nginx:alpine

# แสดงรายการ container ที่กำลังทำงาน
docker ps

# หยุดมัน
docker stop demo

# ลบทิ้ง
docker rm demo
```

```text
CONTAINER ID   IMAGE          COMMAND                  CREATED        STATUS        PORTS     NAMES
a1b2c3d4e5f6   nginx:alpine   "/docker-entrypoint.…"   3 seconds ago  Up 2 seconds  80/tcp    demo
```
