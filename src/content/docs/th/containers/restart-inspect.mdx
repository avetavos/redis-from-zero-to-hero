## นโยบาย restart

โดยค่าเริ่มต้น container ที่หยุดหรือ crash จะอยู่ในสถานะหยุด นโยบาย restart บอก Docker ว่าเมื่อใดควรรีสตาร์ทอัตโนมัติ

| นโยบาย | พฤติกรรม |
|--------|---------|
| `no` | ไม่รีสตาร์ทเลย (ค่าเริ่มต้น) |
| `always` | รีสตาร์ทเสมอ แม้แต่หลัง `docker stop` ด้วยตนเอง เริ่มต้นเมื่อ Docker daemon boot |
| `unless-stopped` | รีสตาร์ทอัตโนมัติ แต่ไม่รีสตาร์ทถ้าคุณหยุดด้วยตนเอง เริ่มต้นเมื่อ daemon boot เว้นแต่ถูกหยุดไว้ |
| `on-failure[:N]` | รีสตาร์ทเฉพาะเมื่อ exit code ไม่ใช่ศูนย์ มี retry count สูงสุดเป็น optional |

```bash
# แนะนำสำหรับ service ที่ต้องทำงานต่อเนื่อง
docker run -d --restart unless-stopped nginx:alpine

# รีสตาร์ทสูงสุด 5 ครั้งเมื่อเกิดข้อผิดพลาด
docker run -d --restart on-failure:5 myworker:latest
```

## การ inspect container

`docker inspect` คืนค่า JSON configuration ทั้งหมดของ container — network settings, mounts, ตัวแปร environment, นโยบาย restart, และอื่นๆ:

```bash
docker inspect webserver
```

ใช้ `--format` พร้อม Go template syntax เพื่อดึงข้อมูลเฉพาะ field:

```bash
# รับ IP address ของ container
docker inspect --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' webserver

# รับนโยบาย restart
docker inspect --format '{{.HostConfig.RestartPolicy.Name}}' webserver
```

## การติดตาม resource แบบ live ด้วย `docker stats`

`docker stats` แสดงตารางที่อัปเดตแบบ live ของ CPU, memory, network I/O, และ disk I/O สำหรับ container ที่กำลังทำงาน:

```bash
# container ทั้งหมดที่กำลังทำงาน
docker stats

# container เฉพาะ, snapshot เดียว (ไม่ update แบบ live)
docker stats --no-stream webserver
```

```text
CONTAINER ID   NAME        CPU %  MEM USAGE / LIMIT   MEM %   NET I/O       BLOCK I/O
a1b2c3d4e5f6   webserver   0.1%   5.2MiB / 7.62GiB    0.07%   1.2kB / 648B  0B / 0B
```

## การจำกัด resource

Container สามารถใช้ memory และ CPU ทั้งหมดที่มีได้ถ้าไม่กำหนดขีดจำกัด ตั้งค่าขีดจำกัดตอนรัน:

```bash
docker run -d \
  --name limited \
  --memory 256m \
  --cpus 0.5 \
  nginx:alpine
```

- `--memory 256m` — จำกัด memory สูงสุด (container จะถูก OOM-kill ถ้าเกินนี้)
- `--cpus 0.5` — อนุญาตให้ container ใช้ CPU core ได้ครึ่งหนึ่ง

ยืนยันว่าขีดจำกัดถูกนำไปใช้:

```bash
docker inspect --format '{{.HostConfig.Memory}}' limited
# 268435456  (256 * 1024 * 1024 bytes)
```

docker run -d --name webserver --restart unless-stopped nginx:alpine

docker inspect --format '{{.HostConfig.RestartPolicy.Name}}' webserver

# 4. Live stats (one snapshot)
docker stats --no-stream webserver

# 5. Run a memory-limited container
docker run -d --name limited --memory 128m --cpus 0.25 nginx:alpine

# 6. Confirm the memory limit
docker inspect --format '{{.HostConfig.Memory}}' limited

# 7. Clean up

<Callout title="ใช้ --restart unless-stopped สำหรับ service ที่ต้องทำงานต่อเนื่อง">
`unless-stopped` คือนโยบาย restart ที่แนะนำสำหรับ service อย่าง web server, database, และ background worker Container จะรอดจาก host reboot และ crash ที่ไม่คาดคิดอัตโนมัติ แต่ยังยอมรับ `docker stop` ที่ตั้งใจ — เพื่อให้คุณสามารถนำออกจากการให้บริการเพื่อ maintenance ได้โดยไม่มีปัญหา
</Callout>
