## การรัน container

`docker run` เป็นคำสั่งที่ใช้บ่อยที่สุด มันจะ pull image ถ้าจำเป็น, สร้าง container, และเริ่มการทำงาน

```bash
docker run nginx:alpine
```

คำสั่งนี้รัน nginx ใน **foreground** — terminal ของคุณจะ attach กับ output ของมัน กด `Ctrl+C` เพื่อหยุด

### โหมด detached ด้วย `-d`

เพิ่ม `-d` เพื่อรัน container ในเบื้องหลัง:

```bash
docker run -d nginx:alpine
```

Docker จะแสดง container ID เต็มและคืน prompt ทันที

### การตั้งชื่อ container ด้วย `--name`

Docker จะกำหนดชื่อแบบสุ่มโดยค่าเริ่มต้น ใช้ `--name` เพื่อตั้งชื่อที่คาดเดาได้:

```bash
docker run -d --name webserver nginx:alpine
```

## การแสดงรายการ container

```bash
# เฉพาะ container ที่กำลังทำงาน
docker ps

# container ทั้งหมด — ทั้งที่กำลังทำงานและหยุดแล้ว
docker ps -a
```

```text
CONTAINER ID   IMAGE          STATUS                     NAMES
a1b2c3d4e5f6   nginx:alpine   Up 2 minutes               webserver
b7c8d9e0f1a2   alpine         Exited (0) 5 minutes ago   old-task
```

คอลัมน์ `STATUS` บอกว่า container กำลังทำงาน (`Up`) หรือหยุดแล้ว (`Exited`)

## การหยุด, เริ่ม, และรีสตาร์ท

```bash
# หยุดอย่างสุภาพ — ส่ง SIGTERM, รอ 10 วินาที, แล้วจึง SIGKILL
docker stop webserver

# เริ่ม container ที่หยุดอยู่
docker start webserver

# รีสตาร์ท (หยุดแล้วเริ่มใหม่)
docker restart webserver
```

`docker stop` **ไม่ได้** ลบ container ออก มันยังคงอยู่บนดิสก์ในสถานะหยุด

## การลบ container

```bash
# ลบ container ที่หยุดอยู่
docker rm webserver

# บังคับลบ container ที่กำลังทำงาน (ส่ง SIGKILL)
docker rm -f webserver
```

คุณไม่สามารถลบ container ที่กำลังทำงานได้โดยไม่ใช้ `-f`

## การทำความสะอาดอัตโนมัติด้วย `--rm`

สำหรับงานที่ทำครั้งเดียวและไม่ต้องการจัดการด้วยตนเอง ให้เพิ่ม `--rm` Container จะถูกลบอัตโนมัติทันทีที่กระบวนการสิ้นสุด

```bash
docker run --rm alpine echo "Hello, Docker!"
```

```text
Hello, Docker!
```

Container หายไปทันทีที่ `echo` เสร็จสิ้น — ไม่ต้องรัน `docker rm` เพิ่ม

</Callout>
