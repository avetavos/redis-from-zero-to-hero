## Named volume คืออะไร?

**Named volume** คือหน่วยพื้นที่จัดเก็บที่ Docker สร้างและจัดการแทนคุณ Docker เก็บข้อมูล volume ไว้ในไดเรกทอรีของตัวเอง (โดยทั่วไปคือ `/var/lib/docker/volumes/` บน Linux) คุณอ้างถึง volume ด้วยชื่อ — ไม่ใช่ด้วย host path

เนื่องจาก Docker เป็นเจ้าของตำแหน่งที่จัดเก็บ named volumes จึง:

- ทำงานได้เหมือนกันบน Linux, macOS และ Windows
- รอดจาก `docker rm` — volume แยกจาก container
- สามารถแชร์ระหว่าง container หลายตัวพร้อมกันได้
- เป็นวิธีที่แนะนำในการเก็บข้อมูลแอปพลิเคชันใน production

## การจัดการ volumes

### สร้าง volume

```bash
docker volume create mydata
```

### แสดงรายการ volumes ทั้งหมด

```bash
docker volume ls
```

ผลลัพธ์ที่คาดหวัง:

```text
DRIVER    VOLUME NAME
local     mydata
```

### ตรวจสอบรายละเอียด volume

```bash
docker volume inspect mydata
```

```text
[
  {
    "CreatedAt": "2024-06-01T10:00:00Z",
    "Driver": "local",
    "Mountpoint": "/var/lib/docker/volumes/mydata/_data",
    "Name": "mydata",
    "Scope": "local"
  }
]
```

`Mountpoint` คือที่ที่ Docker เก็บข้อมูลจริงบน host คุณแทบไม่จำเป็นต้องเข้าถึงโดยตรง — Docker ซ่อนรายละเอียดนั้นไว้

### ลบ volume

```bash
docker volume rm mydata
```

ไม่สามารถลบ volume ได้ขณะที่ container กำลังใช้งานอยู่ ลบ container ก่อน หรือใช้ `docker volume prune` เพื่อลบ volumes ที่ไม่ได้ใช้ทั้งหมดในครั้งเดียว

## การ mount volume เข้า container

มีสองรูปแบบ syntax ทั้งสองเทียบเท่ากัน — ใช้แบบไหนก็ได้ที่คุณอ่านง่ายกว่า

**Short `-v` syntax:**

```bash
docker run -d --name myapp -v mydata:/app/data myimage
```

**Long `--mount` syntax (ชัดเจนและแนะนำสำหรับ scripts):**

```bash
docker run -d --name myapp \
  --mount type=volume,source=mydata,target=/app/data \
  myimage
```

ถ้า volume `mydata` ยังไม่มีอยู่ Docker จะสร้างให้อัตโนมัติเมื่อคุณรัน container

## ข้อมูลคงอยู่ข้ามการสร้าง container ใหม่

คุณสมบัติสำคัญของ named volume: คุณสามารถลบ container และสร้างใหม่ได้ ข้อมูลทั้งหมดที่เขียนลง volume ยังคงอยู่ครบถ้วน

```bash
# รัน container และเขียนข้อมูล
docker run --name writer -v mydata:/app/data alpine sh -c "echo 'persistent' > /app/data/file.txt"

# ลบ container
docker rm writer

# สร้าง container ใหม่ mount volume เดิม
docker run --rm -v mydata:/app/data alpine cat /app/data/file.txt
# Output: persistent
```

docker volume inspect demo-vol
